MethScope

cli

Ultra-fast analysis of sparse DNA methylomes via MRMP (Most Recurrent Methylation Pattern) encoding — cell-type annotation, deconvolution, and genome-wide CpG upscaling, as a single pure-C binary. No R, no Python at runtime.

GitHub· conda· models· YAME· R / MethScope· hg38 · mm10 · WGBS · single-cell · spatial

Install & fetch the models

# conda — installs the `methscope` binary
conda install -c zhou-lab \
  -c conda-forge methscope

# or build from source (bundles YAME)
git clone --recurse-submodules \
  https://github.com/zhou-lab/methscope-cli
cd methscope-cli && make

# pretrained models + test fixtures
git clone \
  https://github.com/zhou-lab/methscope_data

Every model is one self-contained bundle that already carries its MRMP feature definition (and labels), so a query .cg runs directly — no separate annotation files. Examples below run from the methscope_data/ checkout.

mrmp — the MRMP foundation

An MRMP is one recurrent methylation pattern across a reference panel; the mask that labels every genomic CpG with its pattern is what every model is built on. mrmp constructs it natively from a discretized reference .cg — no YAME + awk + sort pipeline, ~576 MB peak.

methscope mrmp build --reference ref.cg \
  --patterns 1000 -o panel.mrmp
methscope mrmp inspect panel.mrmp
# export a YAME .cm mask + tables
methscope mrmp export panel.mrmp \
  --mask panel.cm --patterns pat.tsv

Each pattern packs to a base-3 uint64 key; selection is deterministic (count desc, key asc), so a build is byte-reproducible. The whole-genome unit index for upscale-train reads the artifact directly (_upscale index --mrmp).

predict — cell type / sex

Classify a methylome against a bundled classifier (xgboost / logistic / threshold) → label + confidence per cell.

# cell type (62 human types)
methscope predict test/human_hg38_celltypes.cg \
  models/hg38_celltype.ubjx

# sex (XCI logistic model)
methscope predict test/human_hg38_celltypes.cg \
  models/hg38_sex.ubjx

deconv — cell-type proportions

Non-negative least squares of a mixture .cg against a .refx signature panel → a proportion per cell type. Uses all MRMP patterns; robust to extreme sparsity (split-MRMP references).

methscope deconv test/human_hg38_immune_mixture.cg \
  models/hg38_65celltypes.refx
cell  … Macrophage  Mono    NK_CD56  …
1     … 0.7486      0.2370  0.0000   …

The 65-type reference spans 58 Zhou single-cell + 7 Loyfer organ/blood types for whole-body / cfDNA deconvolution.

upscale — impute genome-wide CpG

Reconstruct dense CpG methylation from a sparse methylome with a bundled decoder (.updecx). Pure-C inference — no CUDA or BLAS at run time.

methscope upscale -o imputed.cg \
  models/hg38_10k1.updecx test/human_hg38_test.cg
[methscope] upscaled 1 sample(s) x 29,401,795 CpGs (genome .cg)

matrix — featurize to a beta matrix

The featurization shared by predict/deconv/train: summarize each query record against every MRMP pattern → a record × pattern beta TSV, or a .refx deconvolution reference with --refx.

# feature matrix (TSV to stdout)
methscope matrix query.cg panel.mrmp > betas.tsv

# build a deconvolution reference
methscope matrix --refx -o panel.refx \
  celltypes.cg panel.cm

train — fit a classifier

Train a label classifier over MRMP features and bundle it with its MRMP + labels in one step. Three frameworks; the bundle records which.

methscope train -l labels.txt \
  -o hg38_celltype.ubjx query.cg panel.cm
# --framework xgboost | logistic | threshold

Linear models store per-feature training means, so missing/sparse betas are imputed at predict time — accurate even at <1% genome coverage.

upscale-train — whole-genome decoder

Train the unified whole-genome upscale model (UPDEC2) on CUDA — one processing unit per MRMP membership. Inference stays pure-C.

make CUDA=1 CUDA_HOME=/path/to/cuda CUDA_ARCH=sm_80
methscope upscale-train -i train.msur \
  --index units.msui --mrmp panel.cm \
  -o model.updecx --work-dir work/

bundle / unbundle — package a model

A bundle wraps an inner model + its MRMP (+ labels) into one self-describing file (.ubjx / .updecx / .refx) so it ships and runs stand-alone.

methscope bundle -m panel.cm -l labels.txt \
  model.ubj -o model.ubjx
methscope unbundle model.ubjx      # -> parts

inspect — look inside a bundle

Report a bundle's framework mark, on-disk section layout, and model summary (labels, weights, feature count).

The MRMP idea — why a query .cg runs against a tiny feature set

DNA methylation is stored positionally as compact YAME .cg files — one M/U or beta value per CpG in a fixed genome order, with no coordinates inside. A Most Recurrent Methylation Pattern (MRMP) mask (.cm, or a .mrmp artifact) labels every CpG with the recurrent cross-sample pattern it belongs to. To featurize a query, MethScope averages the observed CpGs within each pattern → one beta per pattern (a few hundred to a few thousand features), which is what every model consumes. Because the reference panel's rare, cell-type-specific patterns are exactly the informative ones, this stays accurate under extreme sparsity — single-cell and spatial input, or cfDNA.

Featurize once, reuse everywhere

# the shared step behind predict/deconv/train
methscope matrix query.cg panel.mrmp
cell   P1     P2     P3    …  Pna
1      0.91   0.04   NA    …  0.62
# Pna = background; NA = no observed CpG
